其中1103-1.ent意味着这件事1103结束，是1103这天结束的第一件事

event实际上就是聊天记录的储存

每条聊天记录的结构为（基于ent结构

```ent
{
memory_state:state//这是存储的记忆的状态，但是其实存进去以后就没啥大用了，只是保留着这件事情终结的时候的样子，如果本event还在进行，那么这玩意是有用的
current:"..."//这是一个转义后的字符串，记载的是该条信息的聊天记录，如果是文件，这里直接是"^userName/modelName %[fileName.xxx] - .../2023/1/1/MD5CODE.xxx",也就是说聊天记录里用到的文件会经过一个MD5算法后保留后缀作为文件名然后放在同目录下
extraCurrent:...//这里是一个空的对象
TIMESET:"20201105-11:05"//这是附带的时间戳
}
```

event.ts应该负责维护事件的状态，不用RAG检索

一个事件列表大部分应该在本次对话（多轮）开始的时候被加载出来，之后不再进行被动的插入中间，但是可以被动的插入尾部

主动方法待定...(大概是记住某件事之类的，或者RAG检索到的事件被用到了)

R 值、距离时间、重要程度会生成一个事件权重（event_weight），以这个权重决定事件是否被加入

事件权重的计算公式是：
当距离时间小于3天，设Tw = 1

当距离时间小于7天，设Tw = 0.6

当距离时间小于1个月（30天），设Tw = 0.3

当距离时间小于一年（365天），设Tw = 0.1

如果距离时间大于等于一年，则Tw = 0.05



重要程度设为Im(importance)

范围应该是0~10

如果这件事只是普通发生的一件小事，其重要程度应该是1

如果这件事是经过一些思考才得以解决的事，其重要程度应该是2

如果这件事是认真思考解决某问题才完结的事，其重要程度应该是3

如果这件事是略微带有一定情感起伏的事，其重要程度应该是4

如果这件事里有对一些事情未来的想法和规划，其重要程度应该是5

如果这件事让你感到非常开心，其重要程度应该是6

如果这件事是非常带有感情的，其重要程度应该是7

如果这件事深深触动了你自己，其重要程度应该是8

如果这件事深深改变了你，其重要程度应该是9

如果这件事决定了你做事情的基本，决定了你成为一个怎么样的存在，其重要程度是10

如果一件事同时符合以上多个特征，以最大的为准，重要程度允许为小数，重要程度0的事件不会被记下来

重要程度在非常苛刻的条件下是可以重写的，其由LLM评估，但是目前待定，一般情况下其会在事件结束时被写好



R值的取值范围是0~1

那么：event_weight = R * 10 * Im * Tw

最后会根据event_weight排序，输出其前100条的事件（虽然看起来是简介



为了维护事件的简介，同时会有一个event_summary文件夹，需要注意的是event文件夹里的信息主要是指某条具体聊天的信息，而event_summary才是真正的事件信息，也就是记载事件列表时需要用到的信息

其维护的是各个事件的简介，其命名规则和事件文件夹里的一样，唯一不同就是其文件虽然也是ent，但是只存一个条目

```ent
{
memory_state:state//这是存储的记忆的状态,是这一整件事的状态
current:"..."//这是agent看到的实际内容，是时间的简介
extraCurrent:{
	Im:,//这里是该事件的重要程度
	}
TIMESET:"20201105-11:05"//这是附带的时间戳,格式可能不太对，以cyan_time.ts里面的标准全时间为准
}
```
