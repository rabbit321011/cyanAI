### 🔴 高危问题

#### 1.1 XSS 安全风险
- **文件**: `ContextShow.vue`
- **位置**: 模板部分 `v-html`
- **问题描述**: 使用 `v-html` 渲染 Markdown 内容，未进行 XSS 过滤
- **风险**: 如果后端返回恶意 HTML/JS，会被执行
- **建议**: 使用 DOMPurify 清理 HTML
```typescript
import DOMPurify from 'dompurify'
const rawHtml = marked.parse(content) as string
return DOMPurify.sanitize(rawHtml)
```

#### 1.2 硬编码的 API 地址
- **文件**: `globalState.ts`
- **位置**: 第 5-8 行
- **问题描述**: API 地址硬编码在代码中
- **风险**: 
  - 内网地址泄露
  - 部署时需要修改代码
- **建议**: 使用环境变量 `import.meta.env.VITE_API_URL`

#### 1.3 非响应式全局状态
- **文件**: `globalState.ts`
- **问题描述**: 使用普通对象而非响应式对象
- **影响**: 修改 `userName` 后，使用它的组件不会自动更新
- **建议**: 使用 `reactive()` 或 Pinia

#### 1.4 命令解析中异步调用问题
- **文件**: `webOperation.ts`
- **位置**: `commandMessage` 函数
- **问题描述**: `newChat()` 是异步函数但没有 await
- **影响**: 
  - 后端可能还没收到请求就返回成功
  - 请求失败时用户已看到成功消息

#### 1.5 API 响应未处理错误情况
- **文件**: `App.vue`
- **位置**: 第 43-52 行
- **问题描述**: `sendMessage` 返回错误时，错误消息会直接显示给用户
- **建议**: 检查 `isError(sendMessageResponse)` 并显示友好的错误提示

#### 1.6 超时时间过长
- **文件**: `webOperation.ts`
- **位置**: 第 41 行
- **问题描述**: 5 分钟超时太长
- **影响**: 用户会等待很久才能看到错误提示
- **建议**: 30-60 秒更合理
